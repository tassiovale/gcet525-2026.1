# Introdução

## Diferença enter compilador e interpretador

Um compilador é um programa de computador que traduz o código-fonte escrito em uma linguagem de alto nível para outra linguagem, normalmente uma linguagem de máquina ou linguagem de baixo nível. Ele analisa todo o programa de uma vez e gera um código executável. Exemplos de linguagens que utilizam compiladores são C, C++ e Java (geralmente compiladas para código de máquina). Além disso, a execução tende a ser mais rápida, pois não há tradução durante a execução do programa.

Já um interpretador é um programa que lê, traduz e executa o código-fonte linha por linha, sem gerar um executável independente. Por esse motivo, a execução costuma ser mais lenta, já que a tradução e a execução ocorrem ao mesmo tempo. Exemplos de linguagens interpretadas são Python, Ruby e JavaScript (implementações puramente interpretadas).

## Fases da compilação
1. Análise lexical (scanning)
	- converte o código bruto em uma sequência de tokens → ignora comentários e espaços
    - int x = 10; → `[KW_INT][ID(x)][=][NUM(10)][;]` → `[tipo de dado][nome da variável][operador][literal numérico][pontuação]`
2. Análise sintática (parsing)
    - verifica se esses tokens estão organizados de forma correta sintaxicamente
      - caso esteja incorreto, retorna um erro de sintaxe na IDE
    - retorna uma árvore sintática
3. Análise semântica
    - verifica se o código “faz sentido” logicamente
      - verifica erros de declaração de variáveis, tipos e etc
    - retorna uma arvore de sintaxe com mais anotações
4. Geração de código intermediário
    - está entre linguagem de alto nível e baixo nível → linguagem interna do compilador
    - simplifica o processo de otimização executando técnicas como: eliminar código morto, otimizar loops, reutilizar expressões comuns
    - não é específico de nenhuma máquina, podendo ser facilmente migrado
5. Otimização de código
    - aprimora ainda mais o código intermediário para diminuir o tempo de execução
    - pré computa constantes, elimina cópias de variáveis, reaproveita resultados e etc
6. Geração de código
    - é onde a linguagem intermediária é transformada em código de máquina ou assembly 

## Diferença entre pré-processador, compilador, assembler, linker

| Etapa | O que faz | Diferença chave | Entrada | Saída |
|---|---|---|---|---|
| Pré-processador | Processa diretivas (#include, #define, #ifdef), expande macros e remove comentários | Só prepara o texto do código, não entende lógica do programa | Código fonte (.c, .cpp) | Código fonte expandido |
| Compilador | Analisa sintaxe e semântica, otimiza e traduz o código | Entende a linguagem de programação e detecta erros | Código pré-processado | Assembly ou código intermediário |
| Assembler | Converte assembly para código de máquina | Primeiro passo que gera binário para CPU | Código Assembly (.s) | Código objeto (.o, .obj) |
| Linker | Junta arquivos objeto e bibliotecas, resolve símbolos externos | Cria o executável final | Arquivos objeto + bibliotecas | Executável final (.exe, .out) |


## Definir linguagem intermediária
A definição de Linguagem intermediária (LI) é baseada na geração de código abstrato após a análise semântica. Em resumo, a análise semântica que atua como fiscal de qualidade no compilador e após garantir que o código faz sentido, a sintaxe de parênteses, chaves, ponto e vírgula por exemplo, é e descartada para a geração do código abstrato. Dessa forma, nessa etapa o compilador quebra a lógica mas ainda não se preocupa ainda com hardware. A partir do exemplo abaixo é possível esclarecer a ideia de Linguagem Intermediária.

```Exemplo:

Código Fonte
x = a + b + c;

Código Abstrato (Linguagem Intermediária)
variavelTemporaria = a + b;
x = variavelTemporaria + c;
```

> É importante destacar que a abstração é indicada a partir da ideia da "variavelTemporaria" não possuir ao certo endereço, no memória RAM ou no processador. 

### Qual o grande ponto na ideia de existir a Linguagem Intermediária?
O compilador resolve a lógica por código abstrato e no fim traduz essa lógica para o hardware segundo suas limitações físicas.

## Definir JIT compilers
- compilador que gera código de máquina “na hora”, ou seja, durante a execução do código, ao invés de compilar tudo antes de rodar
- exemplos de uso: javascript em seu framework React, que compila enquanto roda a aplicação, atualizando quase instantaneamente durante a execução
